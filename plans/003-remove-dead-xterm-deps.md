## Why this matters

`package.json` declares four xterm-related packages, but only the modern scoped
ones are actually imported anywhere in the codebase:

- **Imported:** `@xterm/xterm@^6.0.0`, `@xterm/addon-fit@^0.11.0`
- **Dead (imported nowhere):** `xterm@^5.3.0`, `xterm-addon-fit@^0.8.0` — these
  are the legacy *unscoped* predecessors of the scoped packages.

The two dead packages add install weight and, more importantly, are a
correctness trap: a future contributor could import the legacy `xterm` (v5) by
autocomplete and silently run a second, older terminal engine alongside the v6
one. Removing them is a pure subtraction with no runtime behavior change.

(Note: this is NOT a "two versions loaded at once" bug today — nothing imports
the legacy packages, so nothing double-loads. It is dead-dependency cleanup.)

## Steps

### Step 1: Prove the legacy packages are imported nowhere

Run the search BEFORE removing anything:

```
rg -n "from ['\"]xterm['\"]|from ['\"]xterm-addon-fit['\"]|import\(['\"]xterm['\"]\)|import\(['\"]xterm-addon-fit['\"]\)|require\(['\"]xterm['\"]\)|require\(['\"]xterm-addon-fit['\"]\)" src apps
```

**Verify**: no matches (rg exits 1). If there ARE matches, see STOP conditions —
the package is live and must not be removed.

### Step 2: Remove the two legacy dependency lines

In `package.json`, delete exactly these two lines from `"dependencies"`:

```json
    "xterm": "^5.3.0",
    "xterm-addon-fit": "^0.8.0",
```

Leave `@xterm/xterm` and `@xterm/addon-fit` untouched.

**Verify**: `grep -nE '"xterm"|"xterm-addon-fit"' package.json` → no matches
(exit 1). `grep -nE '"@xterm/xterm"|"@xterm/addon-fit"' package.json` → still
present (exit 0).

### Step 3: Reinstall and confirm nothing broke

```
pnpm install
```

**Verify**:
- `pnpm install` → exit 0 (lockfile updates to drop the two packages and any
  now-unused transitive deps).
- `pnpm exec tsc --noEmit` → exit 0.
- `pnpm build` → exit 0. The terminal UI (`InteractiveTerminal.tsx`) compiles
  and bundles using the scoped packages only.
