## Why this matters

`pnpm audit` at the planned-at commit reports multiple **high** and **moderate**
advisories. The most important are on `next@15.5.15` — a directly-depended
runtime package — which has Server-Component DoS, SSRF, middleware/proxy bypass,
and reflected-XSS advisories, all patched in the 15.5.16–15.5.18 range. These
matter most because OmniHarness can expose its web UI beyond localhost via the
documented cloudflared remote-access path (see `README.md`). A high-severity
transitive `fast-uri` path-traversal/host-confusion advisory reaches the app
through `@mastra/core › @modelcontextprotocol/sdk › ajv`, and a critical
`vitest` advisory affects the dev toolchain. All three are fixable by version
changes with no source edits.

## Steps

### Step 1: Bump `next` and `eslint-config-next` to the latest 15.5.x

In `package.json`, change the pinned versions so both float to the latest
patched 15.5 release:

- `"next": "15.5.15"` → `"next": "^15.5.18"`
- `"eslint-config-next": "15.5.15"` → `"eslint-config-next": "^15.5.18"`

Keep `next` and `eslint-config-next` on the **same** version line (they are
released together). Do NOT jump to 15.6 or 16 — that is a major/minor jump with
breaking-change risk and is out of scope (STOP condition).

Then install:

```
pnpm install
```

**Verify**:
- `pnpm install` → exit 0.
- `pnpm why next` shows a resolved version `>=15.5.18`.

### Step 2: Add the `fast-uri` override

In `pnpm-workspace.yaml`, add one line to the existing `overrides:` block (keep
alphabetical-ish grouping is not required; just add it inside the block):

```yaml
  fast-uri: ^3.1.2
```

Then re-install so the override takes effect:

```
pnpm install
```

**Verify**:
- `pnpm why fast-uri` → every resolved instance is `>=3.1.2`.

### Step 3: Bump `vitest`

In `package.json` devDependencies, change:

- `"vitest": "^3.1.0"` → `"vitest": "^3.2.6"`

Then install:

```
pnpm install
```

**Verify**:
- `pnpm why vitest` → resolved `>=3.2.6`.

### Step 4: Confirm advisories are cleared and nothing regressed

**Verify**:
- `pnpm audit` no longer lists any `next` advisory, any `fast-uri` advisory, or
  the `vitest` critical advisory. (Other unrelated advisories listed in
  "Out of scope" may still appear — that is expected.)
- `pnpm exec tsc --noEmit` → exit 0 (no new type errors from the bumps).
- `pnpm lint` → exit 0.
- `pnpm build` → exit 0. This is the real gate for a Next.js bump and is the
  most likely place a breaking change would surface. It is slow (several
  minutes); let it finish.
