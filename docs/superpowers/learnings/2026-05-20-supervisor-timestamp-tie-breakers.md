# Supervisor Timestamp Ordering Needs Id Tie-Breakers

**Date:** 2026-05-20
**Context:** OmniHarness supervisor observer, supervisor turn context, execution-event ordering, server replay/list/latest reads
**Symptom:** Several server paths asked for "latest", "recent", or chronological replay using only `createdAt`/`updatedAt`. Equal timestamps can occur when rows are inserted in the same millisecond, and SQLite is not a stable ordering oracle for ties. That can change which supervisor file read is considered latest, which queued message drains first, which checkpoint is recovered, or how CLI/ACP replay orders rows.
**Root Cause:** Some snapshot/frontend paths had already learned to order by `(createdAt, id)`, but supervisor context, observer helpers, handoff synthesis, queued-message draining, recovery reads, CLI/ACP replay, plan/session list endpoints, the all-messages route, planning promotion transcript copying, recovery fork selection, quota incident selection, recovery notice selection, and snapshot-cache pruning still had timestamp-only SQL or timestamp-only in-memory sorts.
**Fix:** Added id tie-breakers to supervisor execution-event reads and explicit `createdAt + id` comparators for supervisor context message/worker in-memory sorting. Extended the same rule to handoff, recovery, queued-message, CLI, ACP, worker-snapshot, planning-refresh, plan-list, all-messages, planning-promotion, fork-recovery message selection, quota incident selection, recovery notice selection, and snapshot cache pruning.
**Verification:** Added source-level ordering guards in `tests/server/supervisor-ordering-source.test.ts` and a `latestRunWorkerEvents` regression in `tests/supervisor/observer.test.ts`. Focused verification includes `pnpm vitest run tests/server/supervisor-ordering-source.test.ts`.
**Prevention:** Any query or sort that means "latest", "recent", "first", or "oldest" must include a deterministic tie-breaker. Timestamp-only ordering is not a cursor, and it is not enough for control-plane decisions.
**Skill/Doc Updates:** Updated `docs/architecture/timing-determinism-audit.md` with the broader server ordering resolved shape. No global skill update was needed because the existing client/server state invariant already requires stable ordering and monotonic cursors.
